Stripe Terminal iOS SDK (Internal)

This private repo contains the Stripe Terminal iOS SDK, which is closed-source.

The public repo contains the pre-compiled framework, an open-source example app, public headers, and autogenerated documentation.

Apps

We have 3 different app targets for the iOS SDK:

Example

This is the open-source example app included in the public Github repo. The project is copied to the public Github repo whenever a new SDK version is released.

AppStore

This is an internal version of the example app – it gets published to internal and external users on TestFlight.

Dev

This is the legacy internal app. It can be used to test features that haven’t made it into the Example or AppStore apps.

Developing in stripe-terminal-ios

Getting started

  • Create a new Apple ID using your @stripe.com email
  • Download Xcode 10.0+
  • Clone this repo
  • > cd stripe-terminal-ios
  • > open StripeTerminal.xcworkspace

Dependencies are checked in to the repo, so you should be able build and run any of the app targets. If you have any issues with dependencies, or add any dependencies, you’ll need to Install Carthage and run the setup script.

  • > brew install carthage
  • > ./setup.sh

For more information on contributing to the SDK and publishing releases, please refer to the wiki.

Publishing a new release

  1. Package a new release:
    • > ./scripts/package_release.rb
  2. Test the generated example app by opening it, adding a key, and running it
  3. Revert any changes to the release folder
  4. Publish a new release:
    • > ./scripts/publish_release.rb
  5. Complete the manual steps printed at the end of the script.
